# MIC Governance

## Project Status

MIC is experimental v0.1.1

The project is currently focused on proving the contract model, MIC Core boundary, engine boundary, and audit model.

## Design Principles

MIC should remain:

- small at the core
- explicit in contracts
- strict about trust boundaries
- auditable by default
- engine-based rather than monolithic
- safe before clever

## Naming

The preferred project name is:

**MIC — Machine Intent Contracts**

Avoid using multiple meanings for MIC in official project material.

## Compatibility Labels

The following labels are reserved for future governance:

- MIC Compatible
- MIC Certified
- MIC Verified
- Official MIC Core
- Official MIC Validator

Do not use these labels without written approval from the project owner.

## Future Direction

Possible future governance work:

- public validator
- approved MIC Core hashes
- engine registry
- engine revocation list
- certification rules
- policy packs
- formal test suite
- conformance profiles
