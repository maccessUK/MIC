What is MIC?

A contract before a sensitive machine action.

MIC describes machine work as a signed, auditable contract.

Instead of allowing automation, AI agents, or backend systems to perform sensitive actions directly, MIC defines what is being requested, what rules apply, what actions are allowed, and what evidence must be produced.

MIC is not an AI model. MIC is not a workflow engine. MIC is a trust layer between intent and execution.

How MIC works

Explicit intent. Bounded execution. Verifiable evidence.

Example use cases

Designed for work where evidence and control matter.

AI

AI Agent Governance

Require AI agents to request approval through a signed contract before performing sensitive actions.

RC

Regulated Change Control

Verify approvals, rollback plans, change windows and audit requirements before system changes are allowed.

CE

Compliance Evidence

Generate structured evidence that required checks were completed before a regulated decision is accepted.
