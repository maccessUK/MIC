# MIC Positioning

## One-Line Description

MIC is a trust and evidence layer for automated task execution.

## Slightly Longer Description

MIC uses signed contracts to define machine intent, allowed actions, execution constraints, and audit requirements before automation is allowed to run.

## What MIC Is

- a contract format
- a MIC Core boundary
- an audit model
- an engine supervision model
- a way to govern automation and AI actions

## What MIC Is Not

- an AI model
- a workflow engine
- a programming language
- a compliance certification by itself
- a replacement for application business logic
- a replacement for legal or regulatory review

## Best Use Cases

MIC is strongest where people need to prove that an automated action was authorised, constrained, and audited.

Good examples:

- AI agent action approval
- regulated change control
- compliance evidence generation
- secure background task execution
- sensitive record access checks
- infrastructure change approval

Bad examples:

- ordinary todo apps
- small scripts
- simple websites
- basic CRUD screens
